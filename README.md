# 台股 AI 資訊儀表板 V20

> V20：以 v19 instant/fast search 為基底，整合事件導向新聞摘要、當日重點具體化，以及背景更新不清空既有畫面。


這是一個可部署的 Next.js 台股觀測平台，整合近即時報價、自選股、台指期、台灣加權指數與近五天新聞。

## 目前主要功能

1. 自選股與個股報價優先讀取 TWSE MIS 近即時行情，並保留盤後日資料作為備援。
2. 前端每 30 秒自動刷新股票報價與台灣加權指數，畫面會顯示更新檢查時間。
3. 台灣加權指數卡片可切換近一周、近一個月、近一年；盤中最新值優先使用 TWSE MIS，歷史日線使用 TWSE 加權指數歷史日線與 Yahoo Finance 備援。
4. 新聞區改為「近五天新聞」，最多顯示 12 則；不足 12 則就顯示實際數量。
5. 台指期區塊保留一般、盤後、合併與多日切換。
6. 中間區塊改為「自選股近期重點」，移除市場排行；個別新聞下方新增「台股熱門排行」，00981A 排第一。
7. 保留 `00631L 元大50正二`、`00878` 等預設自選股與 ETF 歷史資料補強邏輯。
8. 漲跌色彩統一改成台灣盤面習慣：紅色代表上漲，綠色代表下跌。
9. 圖表一般資料點取消顯示，滑鼠移到線上時才顯示較小提示點。

> 注意：TWSE MIS 屬於基本市況報導資料，盤中較接近即時，但仍可能有來源延遲。若要商業公開營運或大量使用行情資料，需確認資料授權。

```bash
npm install

```


## 2. API 路由

### 股票搜尋

```text
GET /api/stocks/search?q=台積電
```

### 全市場行情，預設會嘗試套用 TWSE MIS 近即時資料

```text
GET /api/stocks/quote
```

若想關閉近即時套用，改用盤後/日資料備援：

```text
GET /api/stocks/quote?realtime=0
```

### 多檔自選股近即時行情

```text
GET /api/stocks/quote?codes=0050,00631L,2330,2454
```

### 單一股票行情

```text
GET /api/stocks/quote?code=00631L
```

### 歷史價格

```text
GET /api/stocks/history?code=00631L&range=1m
```

支援：

```text
1w, 1m, 3m, 1y
```

### 台指期 TX 近月日行情

```text
GET /api/futures?session=combined&range=5d
```

支援：

```text
session=regular | after | combined
range=today | 2d | 3d | 4d | 5d
```

### 台灣加權指數

```text
GET /api/market/taiex?range=1w
```

支援：

```text
range=1w | 1m | 1y
```

### 新聞

```text
GET /api/news?code=2330&company=台積電
```

新聞 API 只回傳最近 5 天內的新聞，最多 12 則；若不足 12 則，會直接顯示實際則數，不再用較舊新聞補滿。

### AI 摘要

```text
POST /api/ai/analyze
```

body:

```json
{
  "stock": {},
  "news": []
}
```

## 3. 資料來源設計

目前主要使用：

- TWSE MIS `getStockInfo.jsp`：自選股、個股與加權指數近即時資料
- TWSE OpenAPI：上市公司基本資料、盤後個股日成交資訊
- TWSE `exchangeReport/STOCK_DAY`：個股月成交資料，用來組合歷史趨勢
- TWSE TAIEX 歷史日線與 Yahoo Finance chart API：補足 ETF / 槓桿型 ETF 歷史圖資料，以及加權指數周、月、年日線背景
- TPEx OpenAPI：補上櫃股票清單
- TAIFEX 期貨每日交易行情：台指期 TX 近月日行情
- Google News RSS：新聞標題與原文連結

---

## 4. 重要限制

1. 這不是正式看盤授權版；TWSE MIS 雖較接近盤中行情，但仍可能延遲。
2. 前端 30 秒檢查一次，不代表資料來源一定每 30 秒更新。
3. 台指期區塊目前顯示期交所日行情，不是分時逐筆即時圖。
4. 新聞只保留原文連結與標題摘要，不建議直接複製全文。
5. AI 分析是資訊整理，不是買賣建議。
6. 若要商業公開營運，需確認行情與新聞授權。

## 5. 部署到 Vercel

1. 建立 GitHub repo
2. 把本專案推上 GitHub
3. 到 Vercel 新增專案
4. 選擇這個 repo
5. Environment Variables 可先不填
6. Deploy

若要使用 Groq 做更完整新聞分析：

```bash
GROQ_API_KEY=你的金鑰
GROQ_MODEL=llama-3.3-70b-versatile
```

沒有設定 `GROQ_API_KEY` 時，AI 摘要 API 會直接回傳錯誤；網站不會再使用本地規則式摘要假裝成 AI 摘要。

## V25 Groq API 設定

AI 摘要現在必須使用 Groq API Key。請在 Netlify 的 Environment variables 新增 `GROQ_API_KEY`，scope 需包含 Functions，設定後重新部署。請不要把真實 key 寫進 GitHub 或前端程式。
