# V31 近即時報價補到圖表最後一點

## 修正原因

使用者期待 30 秒更新不只更新報價卡，也能讓個股趨勢圖反映延遲但相對即時的資訊。V30 的圖表只使用日線歷史資料，因此最後可能停在上一個完整交易日，例如 6/12。

## 修正內容

### Python 後端

- 新增 TWSE MIS `getStockInfo.jsp` 近即時報價。
- `/api/stocks/quote?code=0050` 會優先回傳近即時報價，失敗才退回 TWSE 日資料。
- `/api/stocks/quote?codes=...` 也會先抓近即時報價。

### 前端圖表

- 個股圖表仍以 TWSE / Yahoo 歷史日線為主。
- 若選取個股有 TWSE MIS 近即時報價，會把最新價格補成圖表最後一點。
- 只有來源包含 `MIS` 的近即時資料才會補到圖表，不會把 fallback 或盤後日資料硬塞成今天。

## 報告說法

本系統每 30 秒更新近即時報價，並將近即時價格補入個股走勢圖的最後一點，因此使用者可以看到延遲但相對即時的盤中變化；正式歷史日線仍以 TWSE / Yahoo 日資料為基礎。
