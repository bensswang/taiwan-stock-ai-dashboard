# V27 新聞切換保險修正

## 修正重點

### 1. 個股新聞不再顯示上一支股票

這版新增雙層保護：

- `/api/news` 強制 `no-store`，避免 Next/Netlify 快取回傳上一支股票的新聞。
- 後端回傳新聞時，強制把每則新聞標記成目前查詢的股票代號與公司名稱。
- 前端顯示新聞前，再依 `selectedCode` 過濾一次。

因此就算舊請求比較晚回來，或瀏覽器狀態暫時殘留上一支股票的新聞，也不會再把 2330 的新聞卡片顯示在其他股票頁面。

### 2. AI 個股摘要只吃目前股票新聞

「整理可信新聞與當日重點」現在只會送出目前選取股票的新聞，不會把上一支股票殘留的新聞送進 Groq。

## 載入機制說明

個股新聞流程：

```txt
選取股票
→ 清空舊新聞與舊 AI 摘要
→ 呼叫 /api/news?code=股票代號&company=股票名稱&days=5
→ 後端用股票主檔校正公司名稱
→ 抓近五天可信新聞
→ 前端只渲染 code 等於目前 selectedCode 的新聞
```

自選股 AI 摘要流程仍維持 V26：

```txt
12 小時自動整理一次
新增/移除自選股不會立即重算 AI
需要立刻更新時，按頁面上方「重新整理」
```
